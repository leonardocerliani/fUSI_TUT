# Memory Bank - fUSI GLM Analysis Project

## Purpose
This memory-bank serves as the central knowledge repository for the fUSI GLM analysis project. It tracks progress, decisions, technical notes, and resources throughout the development of the Jupyter notebook-based analysis workflow.

## Structure

### Core Documents
- **general_idea.md** - The master project plan outlining all phases and goals
- **project_status.md** - Current progress and status of each phase
- **next_steps.md** - Immediate next actions and priorities
- **decisions.md** - Key architectural and design decisions with rationale

### Technical Documentation
- **technical_notes.md** - Technical discoveries, code patterns, and implementation details
- **data_notes.md** - Information about data structure, formats, and preprocessing
- **glm_notes.md** - Specific notes about GLM implementation, HRF modeling, and contrasts

### Reference Material
- **resources.md** - Links to papers, documentation, tutorials, and examples
- **nilearn_reference.md** - Quick reference for nilearn functions and patterns relevant to the project

## How to Use
1. **Before starting work** - Review project_status.md and next_steps.md
2. **During development** - Update technical_notes.md and data_notes.md as you learn
3. **When making decisions** - Document in decisions.md with reasoning
4. **After completing milestones** - Update project_status.md
5. **When finding resources** - Add to resources.md for future reference

## Project Overview
Building an exploratory, notebook-based workflow to preprocess fUSI data and fit GLMs at the ROI level, with interactive visualization of results. Starting with synthetic data before moving to real data.

**Target**: FSL-style GLM using nilearn for fUSI data analysis
